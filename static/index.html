Export individual formats

NotebookLM sources

Clean, de-timestamped Markdown, one file per lecture, ready to upload to NotebookLM as sources.

Notion study database (CSV)

One row per lecture: Name, Week, Date, Topic, Tags, Status, and Summary. In Notion, choose Import → CSV, then set Tags to Multi-select. When an AI model is configured, the Summary column is generated by the model. Runs as a background job.

Lecture transcripts and subtitles (SRT)

Export transcript text and .srt subtitle files for all recorded lectures. Place the SRT files in the same folder as the video recordings so media players load them automatically.

Anki flashcards Anki-ready

Generates question-and-answer, cloze, and definition cards from the transcripts using an AI model, tagged by course, week, and topic. Produces an Anki-importable .txt file and a .csv file. Requires a configured AI model (see Local AI model above).

Categorize an existing deck — AI-assisted topic tagging

Provide a deck (CSV or tab-separated: front, back[, tags]), or a file path, and the AI model adds course and topic tags. Requires a configured AI model.

Categorization requires an AI model. Start Ollama above, or configure a provider.

Exam cheat sheet (PDF)

Condense the entire course into a dense, exam-ready cheat sheet, generated by the AI model and laid out across a fixed number of A4 pages. The page limit is always respected; lower-value material is omitted to fit. Requires a configured AI model.

An AI model is required. Start Ollama above, or configure a provider.

🎙️ Voice TTS

Convert any Markdown file to spoken audio using Microsoft VibeVoice-Realtime-0.5B. The model runs locally and is downloaded from Hugging Face on first use (~1 GB). GPU strongly recommended — CPU generation is very slow.

VibeVoice is not installed. From the project folder run: pip install -r requirements-tts.txt then restart the app. The model and voices download automatically on first use.
